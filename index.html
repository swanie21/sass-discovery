SASS (Syntactically Awesome Style Sheets)

Steps to learn SASS:

  • Read SASS for Web Designers by Dan Cederholm
  • Make practice files
  • Observe how SASS pre-processes .scss files into .css files

Set-up SASS locally with Command Line

Mac OS X comes pre-installed with Ruby and because SASS is packaged as a Ruby 'gem,' installation is quite simple. In the command line type: gem install sass. After installation you can check the version of your SASS by typing in: sass -v

After SASS is installed we want it to 'watch' files, so whenever we're editing a stylesheet we want SASS to monitor that file and convert the .scss file into a .css file. In the command line type: sass --watch main.scss:main.css. The 'main.scss' is the name of your .scss file that will be converted into the 'main.css' file. These file names can be whatever you prefer them to be named. Typically, the 3 basic file types are module, partial, and vendor files.

SASS files can be organized into directories and the 'watch' command can be used to watch this specific directory. For example, let's say the .scss files are in a 'styles' folder. To watch the changes made in this directory in the command line type: sass --watch styles/main.scss:styles/main.css

SASS files can have different output styles: style expanded, style compact, and style compressed. To specify the output style you can declare it in the command line with the 'watch' command: sass --watch --style expanded main.scss:main.css

Style expanded: line break between declarations
li {
margin-left: 10px;
}

Style compact: each declaration is on one line
li { margin-left: 10px; }

Style compressed: removes all spaces and line breaks
li{margin-left:10px;}

SASS Features:

Variables: allows the user to define and reuse values throughout the stylesheet; needs a $ to specify a value as a variable

$font-sans: 'Lato', sans-serif;

Mixins: allows the user to define and reuse blocks of styles

@mixin example-container {
border: 1px solid #16497E;
display: inline-block;
margin: 0;
padding: 10px;
}

.example {
@include example-container;
}

Mixin Arguments: mixins can take arguments that we pass to the mixin when we call it; user can pass multiple arguments by separating the values with commas in the mixin definition; mixin arguments can be defined as a default but the user can still pass in overrides (@mixin center-container($color, $background: #eee)); mixins are useful for grouping vendor-prefixed rules (-webkit-border-radius, -webkit-box-shadow)

@mixin paragraph-formatting($color) {
color: $color;
font-size: 16px;
}

p {
@include paragraph-formatting(#7e2abf);
}

Nesting: the user can nest CSS rules inside each other instead of repeating selectors in separate declarations; reflects the markup structure

ul {
margin-left: 5px;

li {
color: #1AB78A;
}
}

Nesting Namespaced Properties: the user can nest properties that share a namespace; for example: font-family, font-size, and font-weight can be nested under font

h2 {
font: {
size: 18px;
family: $font-sans;
weight: bold;
}
}

Nesting Media Queries: allows the user to nest media queries inside the original declaration of a selector and it will 'bubble up' into its own separate declaration when the stylesheet is compiled; the user can use variables to define breakpoints

section {
float: left;
width: 65%;

@media screen and (max-width: 800px) {
float: none;
width: auto;
}
}

*Example of using a variable for a breakpoint

$width-large: 800px;
@media screen and (max-width: $width-large) {
float: none;
width: auto;
}

Reference Parent Selector: the user can use an ampersand & as a special placeholder to reference the parent selector when specifying pseudo-classes like hover, focus, etc.

a {
color: #000;
&:hover {
color: #FF0000;
}
}

@import: @import rule allows the importing of multiple .scss files to be merged into a single .css file when compiled; benefits: a single CSS means fewer HTTP connections, variables can be defined in their own file and be imported whenever they are needed, and mixins can be shared and reused through imported .scss files

@import 'reset.scss';
@import 'variables.scss';
@import 'mixins.scss';

@extend: @extend function allows the user to 'chain together' styles that are shared amongst multiple selectors; used when a class has the same styles as another class, except for one or a few other differences; will create multiple, comma-separated selectors for shared styles

.alert {
padding: 15px;
text-align: center;
}

.alert-positive {
@extend .alert;
background: #9c3;
}

Placeholder Selector with @extend: allows the user to define 'phantom' classes that will not appear in the outputted .css file on their own, but are solely for the purpose of extending other styles; the class name is prefixed with a % instead of a period

%button {
padding: 10px;
font-weight: bold;
background: blue;
}

.submit {
@extend %button;
}

Comments: Multi-line comments(/* with forward slashes and asterisks */) are compiled into the final .css output file; Single line comments (// with two forward slashes) are not compiled into the final .css output file

SCSS Formatting

  • List @extend(s) first
  • List @include(s) next
  • List 'regular' styles after that
  • List nested pseudo classes second to last
  • List nested selectors last
  • Never write vendor prefixes
  • Maximum nesting is 3 levels deep and 50 lines
  • Underscores in .scss files indicates the file isn't meant to be compiled by itself

.submit-button {
@extend %button;
@include rounded-corners(5px);
background: Blue;
&:hover {
background: DarkCyan;
}
h3 {
border-bottom: 1px solid #FFF;
}
}
